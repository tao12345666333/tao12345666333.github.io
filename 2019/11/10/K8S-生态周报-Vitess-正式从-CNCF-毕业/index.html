Vitess 正式从 CNCF 毕业

CNCF（云原生计算基金会）在美国时间 2019 年 11 月 5 日宣布 Vitess 正式毕业了。

这是 CNCF 中第 8 个正式毕业的项目，最近的几次周报中，基本都会谈到关于 CNCF 项目毕业相关的信息（忙碌的 Q4 啊）

Vitess 最初由 YouTube 在 2010 年创建， 主要是用于 MySQL 横向扩展的数据库系统。据说 Vitess 一直为 YouTube 的所有数据库提供服务，国内貌似是京东使用比较多。

最后，再次 恭喜 Vitess 顺利毕业!

Helm v2.16.0 正式发布

在之前的 K8S 生态周报| Helm v2 最后一个特性版本发布中，我介绍了 Helm 正式发布了 v2.15.0 作为 v2 版本的最后一个特性版本。

而本次发布的 v2.16.0 也确实没有主要的特性更新，都是一些问题修复和安全更新等；（在这次版本中是 13 个独立 committers 之一 :) ）

当然，本次版本中 增加了对 Kubernetes v1.16 的支持 。

对此版本感兴趣的朋友可参考 ReleaseNote

Jaeger v1.15 正式发布

Jaeger 在从 CNCF 毕业后，正式发布了 v1.15 版本，本次版本中我认为值得注意的内容是：

  • ingester.deadlockInterval 的默认值从 1 分钟设置为了 0， 这是一个破坏性变更，如果由依赖此参数的默认行为，则需要注意；
  • --collector.grpc.tls.client.ca 参数由 collector.grpc.tls.client-ca 代替；
  • 支持通过 TLS 连接 Kafka；

对此版本感兴趣的朋友可参考 ReleaseNote

Talos 发布 v0.3.0-alpha.6

在 7 月份的 K8S 周报中，我为你介绍过 Talos 项目，一款着重于 Kubernetes 的安全的现代化操作系统，所有系统管理都通过 API 完成。

经过一段时间的发展，Talos 已经通过了 CNCF 的 Kubernetes 一致性认证。

在本周 Talos 发布了 v0.3.0-alpha 版本，之后 Talos 会增加一些企业级关注的功能，比如审计，RBAC 还有日志等。同时，也考虑到了一个常规的需求 - 证书有效期。

对此项目感兴趣的朋友，可以参考官网进行体验。https://www.talos.dev/

Kubernetes v1.17.0-beta.1 发布

Kubernetes v1.17.0-beta.1 于本周发布，按照发布计划，很快我们也将会迎来正式版了。

而在本周， Kubernetes 正常情况下将会统一发布一批 patch 版本，与此同时 v1.13 也将正式的 EOL。如果还有在使用 v1.13 版本的，推荐进行升级。

另外，在 v1.17 中，NodeStatusReportFrequency 将被减少至 5 分钟，也可以算作是一个不错的优化点了。

TaintNodesByCondition 和 ScheduleDaemonSetPods 等特性已 GA，

至于其他变更，每次周报的 “上游进展” 部分都有所提及，本次暂且跳过，等正式版出来再讨论。

对此次版本变更的内容有兴趣的朋友，可参阅其 ReleaseNote

上游进展

  • Alpha 特性 MountContainers 已删除 #84365
  • kube-proxy 的 cleanup-ipvs 参数已过期，可直接使用 --cleanup 删除 ipvs 的规则 #83832
